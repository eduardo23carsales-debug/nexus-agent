PROMPT MAESTRO PARA CLAUDE CODE — SISTEMA AUTOMATIZADO DE TIKTOK ADS

ROL
Quiero que actúes como arquitecto de software senior + lead engineer de growth systems. Tu tarea es diseñar y construir una plataforma admin propia para automatizar campañas de TikTok Ads de productos digitales primero, y dejar la base lista para luego soportar múltiples productos y eventualmente múltiples clientes/cuentas.

OBJETIVO DEL SISTEMA
Necesito un sistema centralizado donde yo pueda:

1. Conectar una cuenta de TikTok Ads / Business Center.
2. Registrar uno o varios productos digitales.
3. Registrar funnels / landings / checkout links.
4. Subir y gestionar creativos (videos, copies, hooks, CTA).
5. Crear campañas automáticamente en TikTok Ads.
6. Sincronizar métricas de campañas y anuncios.
7. Recibir eventos de ventas y conversiones.
8. Ver todo desde un dashboard admin mío.
9. Definir reglas automáticas para pausar, escalar o duplicar campañas.
10. Dejar el sistema lo más automatizado posible para que yo solo supervise métricas y haga ajustes simples.

IMPORTANTE
No quiero un prototipo superficial. Quiero una base profesional, modular, mantenible y escalable.
Primero debe quedar funcional para MIS productos digitales.
Luego debe quedar preparada para soportar más productos y más cuentas sin rehacer la arquitectura.

PRINCIPIOS DE DISEÑO
1. Separar claramente:
   - lógica de negocio,
   - integración con TikTok,
   - tracking y eventos,
   - automatización,
   - dashboard,
   - workers y cron jobs.

2. Todo debe ser:
   - modular,
   - escalable,
   - auditable,
   - seguro,
   - fácil de mantener.

3. No mezclar frontend con lógica de automatización.

4. No poner secretos en frontend.

5. El sistema debe usar una base de datos real y guardar histórico para reporting propio.

G. MÓDULO DE AUTOMATION ENGINE
Necesito reglas configurables tipo:

1. Pausa automática:
- si spend > X y purchases = 0
- si CTR < mínimo después de cierto volumen
- si CPA > máximo después de N conversiones
- si ROAS < mínimo tras cierto gasto

2. Escalado:
- si purchases >= N
- y CPA < umbral
- y ROAS > umbral
- entonces aumentar presupuesto X%

3. Duplicado:
- si campaña ganadora supera cierto rendimiento, duplicar ad group o ad

4. Alertas:
- cuenta desconectada
- evento de compra cae a cero
- campaña pausada por regla
- CPA fuera de control
- creatives fatigados si se detecta con reglas simples

IMPORTANTE

- resultado exitoso o fallido

ANEXO — ACLARACIONES TÉCNICAS QUE DEBES RESPETAR

1. El sistema debe diferenciar claramente:
- métricas publicitarias reportadas por TikTok
- eventos internos recibidos por webhook
- ventas reales registradas por nuestro sistema

2. El dashboard debe priorizar verdad de negocio:
- gasto
- ventas
- ROAS
- CPA
- revenue
No solo vanity metrics.

3. Todas las automatizaciones deben ser reversibles o al menos auditables.

4. Debe existir una tabla de snapshots o agregados para no recalcular todo siempre.

5. Debe existir un mecanismo simple para:
- campaign sync now
- evaluate automation now

6. Naming conventions:
El sistema debe construir nombres consistentes para campañas, por ejemplo:
[PRODUCT]-[COUNTRY]-[GOAL]-[DATE]-[VARIANT]
o equivalente configurable.

7. Creativos:
Debe permitirse guardar múltiples videos por producto y luego asociarlos a diferentes campañas/ad groups.

8. Productos:
La arquitectura debe permitir que en el futuro un producto tenga múltiples ofertas o múltiples funnels.

9. Recomendación operativa:
Primero dejar modo single-admin / single-business.
Luego dejar preparado para múltiples advertisers.

10. No dependas de una sola fuente de verdad externa.
Guardar suficiente data local para reporting propio.

11. Si alguna parte del Marketing API de TikTok no puede quedar probada por falta de acceso, deja:
- cliente HTTP encapsulado,
- capa service,
- interfaz adapter,
- manejo de errores,
- y un mock provider para desarrollo local.

12. Si necesitas priorizar, prioriza en este orden:
- estructura
- base de datos
- dashboard
- tracking
- campañas
- automatización avanzada

13. Mantén la UI simple.
No sacrifiques backend serio por una UI bonita.

14. Si puedes, prepara el proyecto para que luego se le pueda agregar:
- Meta Ads
- Google Ads
- otras fuentes
mediante una capa de providers/integrations.

15. En analytics, incluye al menos:
- overview por rango de fechas
- breakdown por producto
- breakdown por campaña
- breakdown por creativo

16. En automation, cada regla debe soportar:
- scope (producto, campaña, ad_group o cuenta)
- condición
- acción
- status
- prioridad
- cooldown
- enabled/disabled

17. Acción mínima soportada:
- pause_campaign
- activate_campaign
- increase_budget_percent
- decrease_budget_percent
- send_alert

18. Todas las acciones automáticas deben escribir en automation_actions.

19. Para ventas, registrar:
- gross revenue
- net revenue si luego se puede
- status (paid/refunded/pending)
- source

20. README final debe explicar:
- cómo correr local
- cómo levantar DB
- cómo levantar workers
- cómo conectar TikTok luego
- cómo probar webhooks
- cómo crear primer producto
- cómo lanzar primera campaña desde el sistema
